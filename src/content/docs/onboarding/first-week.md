Welcome to PF Engineering. One goal for the week: **merge a real PR to production by Friday.** Small is fine. Your buddy will steer you to a good first issue.

## Day by day

- **Mon** — Laptop, accounts, meet your buddy. Run the dev-env setup script; file an issue for anything that doesn't work first try — broken onboarding docs are a SEV3.
- **Tue** — Read this handbook's sections 1–5. Shadow your team's standup. Get `remix-run` running locally (`npm run dev:mocks`) and complete a fake join.
- **Wed** — Pick up your starter issue. Trace one request end-to-end in Datadog — edge to service to database — with your buddy.
- **Thu** — Open your PR. Sit in on an incident review or ops sync — it's the fastest way to learn how production actually behaves.
- **Fri** — Merge. Watch your change move through the [delivery pipeline](/shipping/pipeline/). Post a short "what confused me this week" note in `#eng-onboarding` — we turn those into handbook fixes.

## Access you should have by Tuesday

GitHub (`planetfitness` org), Slack, Jira + Confluence, Datadog, VWO (read), AWS SSO (staging), PagerDuty (observer). Missing something? Your buddy files the access request — don't wait on IT tickets alone.

## Your buddy

Every new engineer gets a buddy for four weeks — a peer, not your manager. Daily 15-minute check-in the first week, then as needed. There is no question too basic. Asking it in a public channel is a feature: the answer helps the next person too.
